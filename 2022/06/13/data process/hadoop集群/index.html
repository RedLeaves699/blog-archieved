    配置的总体思想

    下面的具体过程只是针对我当前的实验环境。如果换了环境，那么下面的操作就没什么参考价值了。不过，配置的总体思想是放之四海而皆准的。

    首先，Hadoop集群是依靠网络来通信的，所以要让所有的机器网络通联。

    其次，Hadoop是用Java写的，它具有良好的跨平台特性。但是它的缺点也很明显，它需要JRE来运行。所以要现在集群的每台机器上配置Java环境。由于Hadoop集群不能使用密码来访问每个节点，所以需要配置SSH来实现主节点对从节点的无密码访问。

    接着，配置Hadoop集群，主要修改几个xml文件，然后将Hadoop设置为环境变量。配置Hadoop有一个小技巧，你只需要配置主机机器的Hadoop，然后直接将配置好的Hadoop目录使用scp命令分发给所有从机即可。

    最后，你就可以使用API接口或命令行来直接操作文件系统了。配置过程也宣告完成。

    起始工作

    特别注意

    如果使用Hadoop，SSH三台机器的用户名必须相同，切记！

    在树莓派上安装操作系统

    在树莓派官网或GitHub下载烧录软件。

    下载完成后，先选择写入的操作系统和储存卡，然后进行设置。设置时要开启SSH服务来实现远程操控终端，设置用户名和密码，配置WIFI来安装必要的程序。其他设置使用默认即可。

    接下来就可以进行烧录了

    这就完成了吗？不，记得将树莓派设为固定ip！ 具体方法是修改cmdline.txt中的文件，在最后加一个空格，写上ip=192.168.168.2。

    使用树莓派搭建Hadoop集群

    更换软件源

    使用apt-get下载时，如果提示空间不足，可以按照如下的方法创建一个软链接：

    sudo mkdir -p "/dev/debs/partial"
sudo rm -rf /var/cache/apt/archives
sudo ln -s "/dev/debs" /var/cache/apt/archives

    使用vim更改软件源，注释所有内容，并添加如下内容。

    下面是需要修改/etc/apt/sources.list文件的内容

    deb http://mirrors.tuna.tsinghua.edu.cn/raspbian/raspbian/ bullseye main non-free contrib rpi
deb [arch=arm64] http://mirrors.tuna.tsinghua.edu.cn/raspbian/multiarch/ bullseye main

    注意，我的开发板系统架构是armv7l的。如果是aarch64，可能有所不同：

    下面的命令用来确定系统架构：

    uname -m

    另外，我们还需要编辑/etc/apt/sources.list.d/raspi.list 文件，删除原文件所有内容，用以下内容取代：

    deb http://mirrors.tuna.tsinghua.edu.cn/raspberrypi/ bullseye main

    全部编辑完后，使用

    sudo apt update

    命令进行更新的确认。

    在整个过程中，终端会有少量报错，不过不影响后续的操作。忽略即可。

    安装OpenJDK

    输入以下命令即可安装：

    sudo apt update
sudo apt-get install default-jdk

    补充：如果按照上面的软连接操作，安装时可能会出现/var/cache/apt/archives/partial is missing.的问题，这是我们只要运行sudo mkdir -p "/dev/debs/partial"即可。

    接着配置 Java 环境变量。编辑/etc/profile文件，插入以下内容：

    # java
export JAVA_HOME=/usr/lib/jvm/java-11-openjdk-armhf/
export PATH=$PATH:$JAVA_HOME/bin

    最后可以使用

    java --version

    验证安装是否成功。

    修改主机名和配置网络映射

    修改主机名 在支持 mDNS(multicast DNS) 的局域网中，使用主机名加上“.local”即 可直接访问主机名对应的设备。例如，树莓派 Raspbian 系统默认的主机名为 raspberrypi， 那么当树莓派接入网络之后，就可以使用 ping raspberrypi.local 来获得树莓派 IP 地址， 当然也可以通过 http://raspberrypi.local 来访问树莓派的 80 端口上的 Web 服务（如果已 经部署了的话）。 修改树莓派的名字涉及两个地方。首先，编辑/etc/hosts，将192.168.168.1后面的名字改为想要的主机名，比如修改为 master

    192.168.168.1 master

    再编辑/etc/hostname，将raspberrypi修改为想要的名字，如 master。

    master

    重复上面的过程修改 slave01 和 slave02 的主机名。

    上述两部修改完的示例如下：

    1. 在/etc/hosts中添加：

      192.168.168.1 master
192.168.168.2 raspberrypi01
192.168.168.3 raspberrypi02
192.168.168.4 raspberrypi03

    2. /etc/hostname：由于安装系统时指定了主机名，所以不必再修改。

    配置 SSH 免密码登录

    首先使用 ssh-keygen 命令产生密钥对，然后通过命令 ssh-copy-id 将登录信息复制到验证文件。

    先在Windows上进行配置，配置之前要在Linux Home创建.ssh文件夹

    Linux

    mkdir ~/.ssh

    Windows

    # 生成密钥
ssh-keygen -t rsa
# 发送密钥
scp C:\Users\Red_Leaves\.ssh\id_rsa.pub redleaves@192.168.168.2:~/.ssh

    然后去树莓派配置

    1. 重命名公钥文件

      mv ~/.ssh/id_rsa.pub ~/.ssh/authorized_keys

    2. 修改文件权限

      chmod 600 ~/.ssh/authorized_keys
chmod 700 ~/.ssh

    3. 修改SSH配置文件

      sudo vim /etc/ssh/sshd_config

    4. 添加语句，保证这三句不被注释掉，没有则手动添加

      RSAAuthentication yes 
PubkeyAuthentication yes 
AuthorizedKeysFile .ssh/authorized_keys

    5. 重启SSH服务

      sudo service sshd restart

    接下来再登录就不再需要密码了。参考：这里

    安装Hadoop

    1. 下载安装包并解压：

      wget https://dlcdn.apache.org/hadoop/common/hadoop-3.3.2/hadoop-3.3.2.tar.gz
tar -zxvf hadoop-3.3.2.tar.gz

    2. 进入到 Hadoop-3.3.2 目录，使用 pwd 命令打印 hadoop 安装路径，并利用鼠标选择复制该路径

      cd hadoop-3.3.2
pwd

    3. 配置Hadoop环境变量并检测是否安装成功

      sudo vim /etc/profile

      插入：

      # hadoop
export HADOOP_HOME=/home/redleaves/Downloads/hadoop-3.3.2
export PATH=$HADOOP_HOME/bin:$HADOOP_HOME/sbin:$PATH
export HADOOP_MAPRED_HOME=$HADOOP_HOME
export HADOOP_COMMON_HOME=$HADOOP_HOME
export HADOOP_HDFS_HOME=$HADOOP_HOME
export YARN_HOME=$HADOOP_HOME
export HADOOP_COMMON_LIB_NATIVE_DIR=$HADOOP_HOME/lib/native
export HADOOP_OPTS="-Djava.library.path-$HADOOP_HOME/lib"
export JAVA_LIBRARY_PATH=$HADOOP_HOME/lib/native:$JAVA_LIBRARY_PATH

      确认环境变量更新：

      source /etc/profile

      直接输入

      hadoop

      如果输出软件命令提示，说明安装成功。

    配置Hadoop

    Hadoop所有的配置文件都在其安装目录的etc文件中，配置时终端要进入Hadoop安装目录。

    配置 core-site.xml 文件

    vim etc/hadoop/core-site.xml

    该配置文件属于 Hadoop 的全局配置文件，我们主要进行配置分布式文件系统 HDFS 的入口地址（即 NameNode 的地址）和 HDFS 运行时所生产数据的保存位置，如下：

    <configuration>
    <!-- 指定 HDFS 中 NameNode 的地址 -->
    <property>
        <!-- 该参数是配置指定 HDFS 的通信地址。其值为 hdfs://BigData01:9000，9000 为端口号。-->
        <name>fs.defaultFS</name>
        <!-- 其中 hdfs 为协议名称，hadoop102 为 namenode 的节点主机名称，端口号为
    9000 -->
        <value>hdfs://raspberrypi01:9000</value>
    </property>
    <!-- 指定 hadoop 运行时产生文件的存储目录，该目录需要单独创建 -->
    <property>
        <!-- hadoop.tmp.dir：该参数配置的是 Hadoop 临时目录，即指定 Hadoop 运行时产生
    文件的存储路径，其值可以自行设置，不能设置为/tmp（/tmp 是 Linux 的临时目
    录）。-->
        <name>hadoop.tmp.dir</name>
        <value>/home/redleaves/hadoop_data/tmp</value>
    </property>
</configuration>

    配置 hdfs-site.xml 文件

    <configuration>
    <property>
        <!-- dfs.namenode.http-address：该参数是配置 NameNode 的 http 访问地址和端口号。
因为在集群规划中指定raspberrypi01 设为 NameNode 的服务器，故设置为 raspberrypi01:50070。-->
        <name>dfs.namenode.http-address</name>
        <value>raspberrypi01:50070</value>
    </property>
    <property>
        <!-- dfs.namenode.secondary.http-address：该参数是配置 SecondaryNameNode 的 http 访问地址和端口号。在集群规划中指定 raspberrypi02 设为 SecondaryNameNode 的服务
器，故设置为 raspberrypi02:50090。-->
        <name>dfs.namenode.secondary.http-address</name>
        <value>raspberrypi02:50090</value>
    </property>
    <property>
        <!-- fs.replication：该参数是配置 HDFS 副本数量。-->
        <name>dfs.replication</name>
        <value>3</value>
    </property>
    <property>
        <!-- dfs.namenode.name.dir：该参数是设置 NameNode 存放的路径-->
        <name>dfs.namenode.name.dir</name>
        <value>/home/redleaves/hadoop_data/dfs/name</value>
    </property>
    <property>
        <!-- dfs.datanode.data.dir：该参数是设置 DataNode 存放的路径-->

    启动和停止集群

    在启动hadoop集群前，需要先格式化NameNode，在Master主机(raspberrypi01)下操作：

    hdfs namenode -format

    格式化后即可启动集群的节点，可以分别启动HDFS和YARN，也可以一起启动：

    全部启动命令：start-all.sh

    启动和停止HDFS：

    start-dfs.sh        #启动HDFS
stop-dfs.sh         #停止HDFS

    启动和停止YARN：

    start-yarn.sh       #启动YARN
stop-yarn.sh        #停止YARN

    全部启动和停止：

    start-all.sh        #启动HDFS和YARN
stop-all.sh         #停止HDFS和YARN

    启动和停止历史（日志）服务器：

    mr-jobhistory-daemon.sh start historyserver     #启动historyserver
mr-jobhistory-daemon.sh start historyserver     #停止historyserver

    最后验证是否启动成功，命令为

    jps

    成功开启后，会看到如下节点：

    终于配置完了。。。

    完成配置后，使用以下的命令查看和创建文件

    hadoop fs -ls /
hadoop dfs -touchz /main.c
hadoop fs -ls

    其他问题

    1. 如果出现permission denied错误，说明主机和从机的SSH连接没有配置好。应该在主机上重新生成SSH，再将公钥分发到从机上。也就是说，生成SSH密钥的过程有两次。第一次是在PC连接树莓派时，第二次是主机连接从机时。

    题目：完全背包

    题目背景

    有\(n\)种物品和一个容量为\(W\)的背包，每种物品都就可以选择任意多个，第\(i\)种物品的价值为\(V[i]\)，体积为\(W[i]\)，求解：选哪些物品放入背包，可使得这些物品的价值最大，并且体积总和不超过背包容量。

    提交链接

    数据规模与约定

    • 对于\(100\%\)的数据，保证\(1 \leq m \le 10^4\)，\(1 \leq t \leq 10^7\)，且\(1 \leq m \times t \leq 10^7\)，\(1 \leq a_i, b_i \leq 10^4\)。

    样例输入

    70 3
71 100
69 1
1 2

    样例输出

    140

    解答

    完全背包问题与01背包问题一样，需要两个维度来描述当前状态，一是现在已经取过哪些物品，二是现在背包还剩下多少空间。

    定义\(f[i][j]\)表示现在已经取过前\(i\)件物品，背包已使用\(j\)的空间，得到的最大价值，则状态转移方程可写为： \[ f[i][j]=\max\{f[i-1][j-k\times W[i]]+k\times V[i]\}\\(0\leq k\leq \lfloor\frac{j}{W[i]}\rfloor)\tag{1.0} \] 式(1.0)与下面的式子等价（等价的条件是\(j\)要按从小到大的顺序枚举）（另外，这个式子是更新公式，所以\(f[i][j]\)要一直在\(\max\)中）： \[ f[i][j]=\max\{f[i][j],f[i-1][j],f[i][j-k\times W[i]]+k\times V[i]\}\\(0\leq k\leq \lfloor\frac{j}{W[i]}\rfloor)\tag{1.1} \] 上面的方程，时间复杂度为\(O(kmt)\)，空间复杂度为\(O(mt)\)。其实上面的方程可以进行优化，我们仔细观察递推式(1.1)与(1.2)： \[ f[i][j-W[i]]=\max\{f[i][j-W[i]],f[i][j-k\times W[i]]+k\times V[i]\}\\(1\leq k\leq \lfloor\frac{j}{W[i]}\rfloor)\tag{1.2} \] 我们发现，其实计算\(f[i][j-W[i]]\)时，就已经计算了\(1\leq k\leq \lfloor\frac{j}{W[i]}\rfloor\)的情况。所以为了避免计算的冗余，我们可以简化递推式，简化得到的递推式如下： \[ f[i][j]=\max\{f[i][j],f[i-1][j],f[i][j-W[i]]+V[i]\}\tag{1.3} \] 时间复杂度降为了\(O(mt)\)，空间复杂度不变。

    因为这个问题是背包的变形，所以它和背包一样，状态只依赖上一个状态和它当前状态已经算过的值，所以可以进行空间的压缩。压缩掉第一维后，空间复杂度降为\(O(t)\)。

    下面的代码中，solve1是上述递推式的直观表达，solve2是solve1的空间压缩优化，solve3是空间压缩后的动态规划过程。

    代码实现

    #define max std::max
#define MAXN (1000 + 10)
typedef long long LL;
// int cnt = 0;
struct obj{
    LL v, w;
}a[MAXN * 10];
LL f[MAXN][MAXN];
LL solve1(int n, int W) {
    LL res = 0;
    for (int i = 1; i <= n; ++i) {
        for (int j = 0; j <= W; ++j) {
            for (int k = 0; k <= j / (int)a[i].w; ++k) {
                f[i][j] = max(f[i][j], f[i - 1][j]);
                f[i][j] = max(f[i][j], f[i][j - (LL)k * a[i].w] + (LL)k * a[i].v);
                //++cnt;
            }
        }
    }
    return f[n][W];
}

LL dp[MAXN * 10000];
LL solve2(int n, int W) {
    LL res = 0;
    for (int i = 1; i <= n; ++i) {
        for (int j = 0; j <= W; ++j) {
            for (int k = 0; k <= j / (int)a[i].w; ++k) {
                // ++cnt;
                dp[j] = max(dp[j], dp[j - (LL)k * a[i].w] + (LL)k * a[i].v);
            }
        }
    }
    return dp[W];
}

LL solve3(int n, int W) {
    for (int i = 1; i <= n; ++i) {
        for (int j = a[i].w; j <= W; ++j) {
            dp[j] = max(dp[j], (LL)dp[j - a[i].w] + a[i].v);
            // ++cnt;
        }
    }
    return dp[W];
}

    运行效率

    1. solve1

      由于内存空间的限制，部分问题不能正确求解，例如数据5。

      如果统计查表次数，直接将计数器写在循环里即可。

    2. solve2

      由于时间复杂度较高，面对较大数据时无法在规定时间内求解。

    3. solve3

      时间复杂度和空间复杂度均符合要求。

    总结

    本题中，时间复杂度的减少，空间复杂度的减少，都是通过对原始递推式的等价变换实现的。对原始递推式的变换可以大大优化时空复杂度。变换递推式是动态规划的优化方法之一。

    2. 继续完成系统启动前的准备工作

    打开计算机电源之后

    Q1 : 程序如何找到引导程序第一条指令的位置？

    A1 : 程序开机时PC/IP的值是由硬件决定的。以 x86 架构为例： x86的机器开机时，CPU 处于实模式（与保护模式相对），实模式的寻址方式是CS:IP方式，即 \((\text{CS}<<4)+\text{IP}\) 。CS是段寄存器，在开机时的初始值为0xffff,PC/IP的初始值为0x0000，这样计算机就可以获得第一条指令的地址0xffff0。这个地址就是 ROM BIOS 映射区的地址，ROM BIOS 实际上是一块flash存储器，存储着当前状态下唯一能够运行的程序。这段程序会先检查RAM，磁盘等设备的状态，检查之后，将磁盘 0 磁道 0 扇区（又称引导扇区）读入 0x7c00 处。然后，设置CS=0x07c0, IP=0x0000，这样计算机接下来就可以执行我们编写的引导程序了。

    Q2 : 引导扇区的作用？

    A2 : 引导扇区就是启动设备的第一个扇区，在实际应用中，按住F1或del键（不同机器略有不同）可进入启动设备设置界面，在这里，可以选择从哪个设备（如光盘，U盘）上读取引导扇区。操作系统的故事从这里开始。。。

    Q3 : 引导扇区代码实现的功能有哪些？

    A3 : 下面是一段引导扇区代码bootsect.s的示例，其中BOOTSEG=0x07c0, INITSEG=0x9000,SETUPSEG=0x9020：

    // 格式8086
//.globl定义一个全局符号，通常是为ld使用。比如经常看到的.global _start
.globl begtext,begdata,begbss,endtext,enddata,endbss 
.text //文本段：一个伪操作符，告诉编译器产生文本段，.text用于标识文本段的开始位置
begtext:
.data //数据段
begdata:
.bss //未初始化数据段
begbss:
entry start //关键字entry告诉链接器“程序入口”
start:
    mov ax, #BOOTSEG mov ds, ax
    mov ax, #INITSEG mov es, ax
    mov cx, #256
    sub si, si sub di, di //两个16位寄存器形成32位寻址ds:si, es:di
    rep movw 
    jmpi go, INITSEG // go是一个地址的标号

    Q3.1 : .text,.data,.bss是什么意思？ A3.1 : 汇编程序中以.开头的名称并不是指令的助记符，不会被翻译成机器指令，而是给汇编器一些特殊指示，称为汇编指示（Assembler Directive）或伪操作（Pseudo-operation）。

    .bss : BSS段（bsssegment）通常是指用来存放程序中未初始化的全局变量的一块内存区域。BSS是英文BlockStartedby Symbol的简称。BSS段属于静态内存分配。 .data : 数据段（datasegment）通常是指用来存放程序中已初始化的全局变量的一块内存区域。数据段属于静态内存分配。

    Q3.2 : rep指令的作用，movw为什么没有源地址和目标地址，jmpi的作用？ A3.2 : rep 指令，表示重复执行该指令后面的汇编代码，执行次数由 cx(16位)，ecx(32位)控制。rep movw实际上是rep movw es:[di] ds:[si](8086)的简写，此外，执行一次movw后，如果再次重复执行，源地址和目的地址都会增加1个字。复制完0x7c00-0x7e00这512个字节后，jmpi 指令令PC/IP=go, CS=INITSEG。虽然上面的代码已经从0x07c00复制到0x90000，但由于jmpi go, INITSEG这个指令，看起来代码还是如同顺序执行一般。

    go: 
    mov ax,cs //cs=0x9000
    mov ds,ax mov es,ax mov ss,ax mov sp,#0xff00
load_setup: //载入setup模块
    mov dx,#0x0000 mov cx,#0x0002 mov bx,#0x0200
    mov ax,#0x0200+SETUPLEN
    //储存在BIOS上的中断，作用是读取磁盘，分析这段代码时要从这里开始
    int 0x13
    jnc ok_load_setup
    mov dx,#0x0000
    mov ax,#0x0000 //复位
    int 0x13
    j load_setup //重读

    Q3.3 : 0x13中断的作用是什么？ h，l分别表示寄存器的高8位和低8位。 0x13是BIOS读磁盘扇区的中断，它的参数由寄存器给定: ah=0x02-表示读磁盘，al=扇区数量(SETUPLEN=4，因为setup占用4个扇区，setup也是操作系统代码重要的组成部分)，ch=柱面号，cl=开始扇区，dh=磁头号，dl=驱动器号，es:bx=内存地址=\(\text(\text{es}<<4)+\text{bx}\)。

    Ok_load_setup: //载入setup模块
    mov dl,#0x00 mov ax,#0x0800 //ah=8获得磁盘参数
    int 0x13 mov ch,#0x00 mov sectors,cx
    mov ah,#0x03 xor bh,bh int 0x10 //读光标
    mov cx,#24 mov bx,#0x0007 // 7是显示属性
    mov bp,#msg1 mov ax,#1301 int 0x10 //显示字符
    mov ax,#SYSSEG //SYSSEG=0x1000，
    mov es,ax
    call read_it //读入system模块
    jmpi 0,SETUPSEG
sectors: 
    .word 0 //磁道扇区数
msg1:
    .byte 13,10
    .ascii “Loading system...”
    .byte 13,10,13,10

    Q3.4 : 0x10中断的作用是什么？ 向屏幕显示字符，bp表示待显示字符的内存地址，cx表示输出字符数量。为了打印字符，我们还需要获取光标信息。参考资料OSdev.org int 0x10

    Q3.5 : 为什么读入system模块还需要定义一个函数? A3.5 : system模块可能很大，要跨越磁道，

    read_it:
    mov ax,es cmp ax,#ENDSEG jb ok1_read
    ret
ok1_read:
    mov ax,sectors
    sub ax,sread //sread是当前磁道已读扇区数,ax未读扇区数
    call read_track //读磁道...
.org 510
    //扇区的最后两个字节，BIOS用以识别引导扇区，否则会打出非引导设备
    .word 0xAA55

    附1：BIOS硬件在内存中映射区0xf0000~0x100000 附2：系统在硬盘（启动盘）上的存储结构

    继续完成系统启动前的准备工作

    移动操作系统主程序

    bootsect.s完成了操作系统的开机引导，接下来OS要完成启动前的设置，接管硬件并依据硬件信息生成对应的数据结构实现管理操作，这是setup.s的工作。 下面是setup.s的代码：

    start:
    mov ax,#INITSEG mov ds,ax mov ah,#0x03
    xor bh,bh int 0x10//取光标位置dx mov [0],dx
    mov ah,#0x88 int 0x15 mov [2],ax ... // 将扩展内存的大小保存到[(CS<<4)+2]中
    cli //不允许中断
    mov ax,#0x0000 cld
do_move:
    // 将操作系统移到0地址，之前腾出0x07c00的空间就是为了让系统有足够的空间放置。
    mov es,ax add ax,#0x1000
    cmp ax,#0x9000 jz end_move
    mov ds,ax 
    sub di,di sub si,si
    mov cx,#0x8000
    rep movsw
    jmp do_move

    Q4.1 : 0x15中断的作用？ A4.1 : 0x15获取物理内存大小并存放在ax寄存器中，参数为ah。

    附3：操作系统硬件参数设计存放地址

    内存地址 长度 名称
    0x90000 2 光标位置
    0x90002 2 扩展内存数
    0x9000C 2 显卡参数
    0x901FC 2 根设备号

    进入保护模式

    在实模式中，依赖 cs:ip 的寻址方式只能访问20位地址（1M），无法满足机器运行的需要。我们需要切换保护模式。

    保护模式不再使用 cs:ip 的方法寻址。x86处理器使用GDT表，GDT表指定了程序段的基址，以及该段的其他属性，使用段内偏移PC/IP+段基址，就可实现32位内存地址访问，即 \(\text{ADDR} = \text{GDT}[\text{CS}]+\text{IP}\)，公式里的CS又被称为选择子。如下图：

    系统中是不存在GDT表的，所以先要初始化GDT表，然后写入正确的内容，x86的GDT表是由硬件实现的。其结构如下：

    在此之后，我们操作32位寄存器（CR0）进入保护模式，CR0的最高位为PG，PG=1时启动分页，最低为是PE，PE=1时启动保护模式。

    下面的代码可以让CPU进入保护模式来访问32位的地址（4G）。

    end_move:
    mov ax,#SETUPSEG
    mov ds,ax
    lidt idt_48 lgdt gdt_48 //设置保护模式下的中断和寻址进入保护模式的命令...
    call empty_8042 
    mov al,#0xD1 out #0x64,al //8042是键盘控制器，其输出端口P2用来控制A20地址线
    call empty_8042 
    mov al,#0xDF out #0x60,al // 选通A20地址线
    call empty_8042 // 初始化8259(中断控制) 
    //一段非常机械化的程序，直接操作寄存器
    mov ax,#0x0001 mov cr0,ax
    // 下面是32位指令了，CS=8 ->(GDT)0x0，程序将从系统所在的0x0地址开始运行
    jmpi 0,8 
empty_8042:
    .word 0x00eb,0x00eb
    in al,#0x64
    test al,#2
    jnz empty_8042
    ret
idt_48:
    .word 0 .word 0,0 //保护模式中断函数表
gdt_48:
    .word 0x800 .word 512+gdt,0x9
gdt: 
    .word 0,0,0,0
    .word 0x07FF, 0x0000, 0x9A00, 0x00C0
    .word 0x07FF, 0x0000, 0x9200, 0x00C0

    在保护模式下继续进行操作系统初始化

    下面简要看一下head.s的代码，这段代码是32位汇编，使用 AT&T 格式，目的是初始化硬件设备（如中断向量，页表等）：

    idt_48:
    .word 0 word 0,0 // 现在忽略中断(向量表)
_idt:
    .fill 256,8,0
stratup_32:
    movl $0x10,%eax mov %ax,%ds mov %ax,%es
    mov %as,%fs mov %as,%gs //指向gdt的0x10项(数据段)
    lss _stack_start,%esp //设置栈(系统栈)
    call setup_idt
    call setup_gdt
    xorl %eax,%eax
1:
    incl %eax
    movl %eax,0x000000 cmpl %eax,0x100000
    je 1b //0地址处和1M地址处相同(A20没开启)，就死循环
    jmp after_page_tables //页表，什么东东?
setup_idt:
    lea ignore_int,%edx
    movl $0x00080000,%eax movw %dx,%ax
    lea _idt,%edi movl %eax,(%edi)

    使用head.s完成操作系统硬件初始化（设置页表除外）后，执行after_page_tables函数跳转到main函数中。

    after_page_tables:
    pushl $0 pushl $0 pushl $0 pushl $L6
    pushl $_main jmp set_paging
L6: 
    jmp L6
setup_paging: 
    设置页表
    ret

    这段程序的控制流如下： 1. 执行after_page_main，将0，0，0压栈作为_main函数的参数，然后将_main的地址压栈。 2. 跳转至设置页表的函数setup_paging，执行 3. 执行完setup_paging后，执行ret指令，读取栈顶地址并赋值给PC/IP，跳转至_main函数。

    Q5 : 为什么向栈内压入一个L6地址？ A5 : 一个操作系统程序的_main函数永远不应该返回。如果_main返回了，就要执行ret指令，PC/IP就要跳转到L6这个位置，L6这个位置的指令是一个死循环，标志着系统运行时出现了问题。

    调用main()函数后，我们在今后就可以使用高级语言编写操作系统了！下面是main函数init/main.c的实现。

    void main(void) { 
    mem_init();
    trap_init();
    blk_dev_init();
    chr_dev_init();
    tty_init();
    time_init();
    sched_init();
    buffer_init();
    hd_init();
    floppy_init();
    sti();
    move_to_user_mode();
    if(!fork()){init();}
}

    // addr:0x90002 就是end_mem的来源
void mem_init(long start_mem,long end_mem) {
    int i;
    for(i=0; i<PAGING_PAGES; i++)
        mem_map[i] = USED;
    i = MAP_NR(start_mem);
    end_mem -= start_mem;
    end_mem >>= 12; // 4k为一页~
    while(end_mem -- > 0)
        mem_map[i++] = 0;
}

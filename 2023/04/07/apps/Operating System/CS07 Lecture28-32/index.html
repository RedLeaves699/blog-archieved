    生磁盘（Raw Disks）的使用

    核心：怎么让磁盘工作起来

    工作过程概览：CPU 向磁盘控制器中的寄存器读写数据，然后，磁盘控制器完成磁盘数据放入内存的工作，并向 CPU 发中断信号。

    程序如何直接使用磁盘 IO

    读写过程：移动磁头，旋转磁盘，读写给定磁道(head)和扇区(sect)上的信息。所以一个程序要读取磁盘信息，只要往控制器中写柱面，磁头，扇区，缓存位置即可。

    void do_hd_request() {
    ...
    hd_out(dev, nsect, sec, head, cy1, WIN_WRITE, ...);
    port_write(HD_DATA, CURRENT->buffer, 256);
}
void hd_out(drive, nsect, sec, head, cy1, cmd, ...) {
    port = HD_DATA; // 数据寄存器端口(0x1f0)
    outb_p(nsect, ++port);outb_p(sect, ++port);
    outb_p(cy1, ++port);outb_p(cy1 >> 8, ++port);
    outb_p(0xA0 | (drive << 4) | head, ++port);
    outb_p(cmd, ++port);
}

    操作系统通过盘块号读写磁盘

    操作系统磁盘驱动负责从 block 号计算出 cy1（柱面磁道号）, head（激活哪个盘面）, sec（扇区）(C,H,S)。磁盘访问时间=写入控制器时间+寻道时间+旋转时间+传输时间，其中的大头是寻道时间。一个程序使用的数据一般是连续的，所以为了减少磁盘的寻道时间，block 在磁盘上的编址也要尽量连续。

    此外，从扇区到盘块的映射可以增加磁盘读写效率。单纯使用硬件，每次只能读写一个扇区，磁盘访问时间约 10ms，寻道时间就占 8ms。如果我们每次寻道时多读取一些内容，那么就可以省去一些寻道的时间。操作系统可以增加读写盘块的大小，使得磁盘可以在旋转时连续读取多个扇区。虽然空间利用率会因此下降（一个块在操作系统中是一个整体，读写区域的大小可能小于块大小），但读写速度可以得到提升。

    编址与\((C,H,S)\)信息的转化关系如下：

    \[ \text{block}=C\times(\text{Heads}\times\text{Sectors})+H\times\text{Sectors}+S \\ \begin{cases}S=\text{block} \% \text{Sectors} \\ H=(\text{block}-S)/ \text{Sectors} \% \text{Sectors} \\ C=(\text{block}-S-H \times \text{Sectors})/ \text{Sectors} / \text{Heads} \end{cases} \]

    static void make_request() {// 处理磁盘请求
    struct request *req;
    req = request + NR_REQUEST;
    req->sector = bh->b_blocknr << 1;// 扇区号是盘块号左移一位，所以一个盘块，2个扇区
    add_request(major + blk_dev, req);
}

    多个进程通过队列使用磁盘

    多进程访问单磁盘，就要涉及到访问调度问题。对于磁盘访问任务，一个好的调度策略应该使磁盘的平均访问时延最小。磁盘访问时间开销的大头是寻道时间，所以磁盘调度时，我们更关心寻道时间。

    我们从 FCFS（先入先服务）调度策略开始：

    磁头开始位置 53，请求队列

    98,103,37,122,14,124,65,67

    问题：寻道跨度大

    SSTF：短寻道优先（移动到离我最近的请求）

    问题：总在中间位置移动，会导致远请求饥饿

    SCAN：阶梯算法（回折）

    问题：中间位置的请求占便宜，因为中间位置的请求被访问两次

    C-SCAN：电梯算法（SCAN+直接移动到另一端）

    问题：实际使用的方法，减少寻道跨度，避免饥饿。

    #define IN_ORDER(s1, s2) \
((s1)->dev < (s2)->dev || ((s1)->dev == (s2)->dev \
&& (s1)->sector < (s2)->sector)) // 根据柱面号排序
static void add_request(struct blk_dev_struct *dev, struct request *req) {
    struct request *tmp = dev->current_request;
    req->next=NULL;
    cli(); // 关中断，request队列是多个进程共享的，必须互斥操作
    for (; tmp->next;tmp=tmp->next)
        if ((IN_ORDER(tmp, req) || !IN_ORDER(tmp, tmp->next)) && IN_ORDER(req, tmp->next)) break; // 放入时应该满足电梯算法的条件
    req->next=tmp->next; // 插入队列
    tmp->next = req;
    sti();
}

    总结：生磁盘的使用

    1. 进程得到盘块号（从文件得到），算出扇区号
    2. 用扇区号make_req（同时申请内存缓冲），用电梯算法add_request
    3. 进程等待
    4. 磁盘中断处理
    5. 唤醒进程，do_hd_request算出 cyl, head, sector.
    6. hd_out调用outp(...)完成端口写

    如何由生磁盘到文件

    形成索引

    让用户使用生磁盘给用户带来的学习成本很高，用盘块号访问磁盘本来就很反直觉。因为用户眼中的文件应该是有意义的一段字符流，而磁盘上的的文件是由盘块上一块块的 0/1 组成的。

    从生磁盘到文件的过程，就是形成字符流到盘块集合映射关系的过程。

    为了增加磁盘读取效率，我们希望文件在磁盘中都是连续存放的。为了读取一个文件，我们只需要直到它在磁盘中的起始位置和占用块数。这个信息是 FCB(File Control Block)中的一部分。

    我们以读取文件中第 200~212 个字符为例，假定一个盘块上的大小为 100B，如果想找到文件对应的盘块号\(\text{P}_b\)，首先应该找到文件的头部 \(h\) 在哪个块（\(\text{H}_b =6\)）中，然后，用当前位置（\(p=200\)）除以块大小\(\text{BLOCK}\)获得当前位置是文件头部块加多少（\(\text{P}_b=\text{H}_b +\lfloor \frac{p}{\text{BLOCK}}\rfloor=8\)）。最后，为了避免索引超出文件大小，我们要对访问作出限制，所以应该指示文件的大小。

    文件连续存放对应的映射关系和 FCB 对应数据结构如下图所示：

    不过，就像数据结构中线性存储一样，对于一些需要经常更改的文件，使用连续存储的效率不高。所以，有的文件是链式存储的。链式存储的概念图如下：

    但链式结构的不支持随机存取方式，读取文件某个部分对应的盘块号很麻烦。顺序结构和链式结构各有优劣，有没有别的方法？

    索引结构

    索引结构利用一个单独的磁盘块存放索引。一个索引中存放着一个文件每一个部分对应磁盘块的位置。如图所示：

    test.c文件的索引块号为 19，文件第 0 个块对应 9 号磁盘块。第 200~212 个字符应该对应文件的 2 号块，文件 2 号块对应 10 号磁盘块。索引结构组合了顺序结构和链式结构，使得块数量的增减和读取更加方便。索引结构是操作系统文件结构的基础结构。操作系统中使用多级索引结构来表示大文件。

    磁盘使用文件的实现

    Linux 将其文件索引结构称为 inode。如果inode中存放文件信息，则inode中存放文件索引表，如果inode存放设备信息，则inode存放主次设备，设备种类等设备信息。

int _bmap(m_inode *inode, int block, int create) {
    if(block < 7) { // 直接块
        if(create && !inode->i_zone[block]){
            inode->i_zone[block] = new_block(inode->i_dev);
            inode->i_ctime = CURRENT_TIME;
            inode->i_dirt = 1;
        }
        return inode->i_zone[block];
    }
    block-=7;
    //进入一级索引，索引占用一个盘块（1KB），每个索引项占用2B，共512个块

    目录结构的实现思路

    为了实现目录结构，磁盘块应该存放什么信息？一个直观的想法是存储目录下所有文件的 FCB，在读取时逐个核对找出当前文件。但 FCB 并不小，而且随文件存在磁盘上，大量的 FCB 读入效率非常低。所以应该存储指向 FCB 的指针。这种想法的概念如下：

    设置一个存放指向目录信息的指针的数据结构（Linux 中称为 i 结点，inode），当我们访问一个目录时，在磁盘数据区寻找目录的信息。目录信息中要么是文件 FCB 的指针，要么是子目录信息，如果我们访问子目录，那么就回到 inode 中再查找。

    总结：磁盘使用过程

    用户：读写文件一部分

    操作系统：

    1. 根据文件目录读取文件的 FCB
    2. 根据 FCB 找到文件块号，然后找到文件对应磁盘块号
    3. 操作系统创建读取请求，将请求写入电梯队列
    4. CPU 产生磁盘中断
    5. 请求被磁盘控制器获得
    6. 中断处理，进程调度

    硬件：磁盘控制器驱动机械装置读写盘上数据

    if((c = get_fs_byte(pathname)) == '/') {//从根目录开始解析
        // 开机启动时，操作系统会自动挂载磁盘，硬盘根目录自动挂载到current->root
        inode=current->root; pathname++;

    while(1) {
        if(!c) return inode; //函数的正确出口
        bh=find_entry(&inode, thisname, namelen, &de);//从目录中读取目录

    }
}

    下面是iget()函数的实现：

    // func : 根据目录项读取下一层inode
struct m_inode* iget(int dev, int nr) {
    struct m_inode * inode = get_empty_inode();
    inode->i_dev=dev;
    inode->i_num=nr;
    read_inode(inode);
    return inode;
}
static void read_inode(struct m_inode *inode) {
    struct super_block *sb=get_super(inode->i_dev);;
    lock_inode(inode);
    // 计算inode块的位置，引导块和超级块各一个+2，再加上i节点位图和盘块位图
    // 大小s_imap_blocks，s_zmap_blocks。inode是从1开始的（0被保留），
    // 所以计算偏移时要-1，然后除一个块中的inode数量。
    block=2+sb->s_imap_blocks+sb->s_zmap_blocks+
        (inode->i_num-1)/INODES_PER_BLOCK;
    bh=bread(inode->i_dev,block);
    // 当前inode在盘块的哪个位置
    inode=bh->data[(inode->i_num-1)%INODES_PER_BLOCK];
    unlock_inode(inode);
}

    find_entry()函数与iget()函数实现的想法差不多。

    #define NAME_LEN 14 // 文件名最大长度
struct dir_entry{
    unsigned short inode; //i节点号
    char name[NAME_LEN]; //文件名
}

// 在fs/namei.c中
#define BLOCK_SIZE 1024 //两个扇区
static struct buffer_head *find_entry
(
    struct m_inode **dir, char *name, ...,
    struct dir_entry **res_dir
) {
    int entries=(*dir)->i_size / (sizeof(struct dir_entry));
    int block=(*dir)->i_zone[0];
    *bh=bread((*dir)->i_dev, block);
    struct dir_entry *de = bh->b_data;
    while(i < entries) {//entries是目录项数
        //读入下一块上的目录项继续match
        if((char*)de >= BLOCK_SIZE+bh->b_data) {
            brelse(bh);
            // 有些项在直接索引，有些在间接索引，所以调用bmap
            block=bmap(*dir,i/DIR_ENTRIES_PER_BLOCK);
            bh=bread((*dir)->i_dev,block);
            de=(struct dir_entry*)bh->b_data;
        }
        if(match(namelen,name,de)) {
            *res_dir=de;
            return bh;
        }
        de++; i++;
    }
}
